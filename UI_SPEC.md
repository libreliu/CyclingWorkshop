# CyclingWorkshop UI 规范

## 1. 整体风格

- **设计语言**：深色主题（暗色背景 + 亮色数据），契合运动/户外场景
- **主色调**：暗灰 `#1a1a2e` 背景 + 亮绿 `#00d4aa` 强调色 + 白色文字
- **字体**：系统字体栈（`-apple-system, "Segoe UI", Roboto, sans-serif`），数据数字用等宽字体
- **布局**：左侧导航 + 右侧主工作区，响应式但主要面向桌面端

---

## 2. 页面结构

采用 **Step-by-Step 工作流** + 自由编辑的双模式设计。

### 2.1 顶栏

```
┌──────────────────────────────────────────────────────────────┐
│ 🚴 CyclingWorkshop          [项目名称]    [导入] [导出] [设置] │
└──────────────────────────────────────────────────────────────┘
```

- 左侧 Logo + 项目名称
- 右侧操作按钮：导入已有项目、导出配置、全局设置（ffmpeg 路径等）

### 2.2 工作流步骤导航

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│ ① 数据源 │──▶│ ② 时间同步│──▶│ ③ 叠加设计│──▶│ ④ 渲染导出│
└─────────┘   └─────────┘   └─────────┘   └─────────┘
```

用户可以随时切换步骤，不必严格按顺序。步骤之间数据自动保存。

## 3. 各步骤 UI 详设

### 3.1 Step ① 数据源

**左栏：FIT 文件**

```
┌─────────────────────────────────┐
│ 📂 FIT 运动数据                  │
│                                 │
│  本地路径:                       │
│  [D:\DCIM\Zepp20260404075746.fit]│
│  [加载]                          │
│                                 │
│ ─ ─ ─ 已加载 ─ ─ ─              │
│ 🏃 骑行 | 2026-04-04 07:38      │
│ 距离: 82.3 km | 时长: 3:12:45   │
│ 均速: 25.6 km/h | 爬升: 1240m   │
│                                 │
│ [查看详情]                       │
└─────────────────────────────────┘
```

- 输入本地绝对路径后点击「加载」，后端直接读取该路径并解析
- 路径不存在时显示红色错误提示
- 解析后自动展示摘要
- 「查看详情」展开面板：逐圈数据、数据字段可用性检测（✅心率 ✅踏频 ✅功率 ❌温度）

**右栏：视频文件**

- 延时倍率下拉：1x, 2x, 4x, 10x, 15x, 30x, 自定义
- 视频预览：弹窗内展示提取的视频帧（后端按时间点抽帧返回图片）

### 3.2 Step ② 时间同步

**核心任务**：将视频时间轴与 FIT 时间轴对齐。

```
┌─────────────────────────────────────────────────────────────┐
│ ⏱️ 时间同步                                                   │
│                                                             │
│ FIT 时间范围: 07:38:00 ─────── 10:50:45                     │
│ 视频时长:     00:00 ─────────────────── 45:23               │
│                                                             │
│ 对齐方式:  ○ 自动推断  ○ 手动偏移  ○ 关键帧对齐              │
│                                                             │
│ ┌── 手动偏移 ──────────────────────────────────────┐        │
│ │ 视频第 [  0  ] 秒 ← 对应 → FIT 第 [  0  ] 秒    │        │
│ │                                                  │        │
│ │ 时间缩放: [1.0]x                                │        │
│ │ (1.0 = 实时, 30.0 = 30倍延时)                    │        │
│ └──────────────────────────────────────────────────┘        │
│                                                             │
│ ┌── 预览验证 ──────────────────────────────────────┐        │
│ │                                                  │        │
│ │  [视频播放器]          [FIT 轨迹 + 时间标记]       │        │
│ │   ▶ 00:15.3            📍 07:38:15               │        │
│ │                                                  │        │
│ │  同步播放: 视频 ▶ 时 FIT 轨迹同步移动              │        │
│ └──────────────────────────────────────────────────┘        │
│                                                             │
│ [← 上一步]                              [下一步 →]          │
└─────────────────────────────────────────────────────────────┘
```

**关键帧对齐模式**（高级）：

```
┌── 关键帧对齐 ──────────────────────────────────────┐
│                                                    │
│ 在视频和轨迹上分别标记 2+ 个同一位置的点：            │
│                                                    │
│ #  │ 视频时间    │ FIT 时间        │ 位置          │
│ 1  │ 00:05:23   │ 07:43:21       │ [地图标点]     │
│ 2  │ 00:22:10   │ 08:01:05       │ [地图标点]     │
│                                                    │
│ [添加关键帧]     计算偏移: +3.0s  缩放: 1.0x       │
└────────────────────────────────────────────────────┘
```

- 用户在视频播放器中 seek 到某个位置 → 在地图上找到同一位置 → 记录配对
- 至少 2 个点可计算 offset + scale

## 4. 组件设计规范

### 4.1 颜色系统

```
背景色:
  --bg-primary:    #1a1a2e    主背景
  --bg-secondary:  #16213e    卡片/面板背景
  --bg-tertiary:   #0f3460    输入框/选中项背景

强调色:
  --accent:        #00d4aa    主强调（绿）
  --accent-hover:  #00f0c0    
  --danger:        #ff4444    警告/错误
  --warning:       #ffaa00    提醒

文字色:
  --text-primary:  #ffffff    主文字
  --text-secondary:#a0a0b0    次要文字
  --text-muted:    #606070    禁用/提示

数据色（Widget 渲染用）:
  --data-speed:    #00d4aa    速度 - 绿
  --data-hr:       #ff4444    心率 - 红
  --data-cadence:  #4488ff    踏频 - 蓝
  --data-power:    #ffaa00    功率 - 橙
  --data-alt:      #aa88ff    海拔 - 紫
  --data-track:    #ffffff    轨迹 - 白
```

### 4.2 叠加层 Widget 视觉规范

Widget 在视频上的渲染应遵循：
- **半透明背景**：默认 `rgba(0, 0, 0, 0.4)` + `border-radius: 8px` + 少量 padding
- **数字字体**：等宽、粗体、白色、投影 `0 1px 3px rgba(0,0,0,0.8)`
- **单位文字**：比数字小 40%，颜色降低透明度
- **圆弧表盘**：270° 弧度，背景弧 `rgba(255,255,255,0.15)`，数据弧渐变色
- **地图**：圆角裁剪，轨迹线 3px，当前位置标记 8px 圆点 + 外发光

### 4.3 交互规范

- **拖拽**：Widget 拖拽时显示虚线边框 + 位置坐标
- **缩放**：Widget 角落拖拽时保持宽高比（可选）
- **选中**：蓝色虚线边框 + 四角缩放手柄
- **时间轴**：拖动滑块时实时更新叠加层预览（节流至 100ms）
- **渲染预览**：点击「预览当前帧」按钮，后端渲染单帧返回 PNG 图片

---

## 5. 响应式与性能

- **最低分辨率**：1280×720
- **推荐分辨率**：1920×1080
- 画布区域按视频宽高比缩放显示
- 大视频文件不做上传，后端直接读取本地路径，仅抽帧预览
- 地图瓦片按需加载，低 zoom 优先

---

## 6. 国际化

- 默认中文 UI
- 数据单位默认公制（km/h, m, ℃）
- 可切换英制（mph, ft, ℉）
- Widget 内单位标签跟随全局设置
